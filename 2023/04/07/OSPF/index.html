    2.1 概述

    开放式最短路径优先协议
    IGP协议
    LS链路状态协议 (距离矢量协议)。
    组播地址：224.0.0.5(都监听)/224.0.0.6(DR和BDR监听)。DROthers通过.6发送LSA给DR，BDR；DR通过.5发送LSA给DRothers。
    Layer2|IPV4|OSPF|FCS (协议号89); OSPFv2 IPv4; OPSFv3 IPv6
    运营商都是使用双栈(Dual Stack)的，即同时支持IPv4和IPv6。
    iSPF增量型SPF（骨干和树枝变更，路由器节点互联是树干，路由器互联网络的接口是树枝）；PRC部分路由计算（路由变更，即树叶）

    1. LSA

    1类LSA中包含RID(路由器ID，应域内唯一)：在区域内的路由器都可以收到，通过LSA泛洪扩散。
    3类LSA包含区域间路由条目，由ABR产生，可以进行汇总(域间汇总)后再传递。

    2.2 OSPF基本运行方式与三张表

    1. OPSF邻居表（邻接关系数据库）：存放着建立邻居路由器的RID，直连的接口地址相关信息。hello周期10S或者30S，dead时间等于四倍hello时间。

    2. LSDB拓扑表（链路状态数据库）：存放各类LSA

    3. 路由表：使用SPF（最短路径优先算法），根据LSA，将自身做树根，计算出一棵cost(开销，带宽)最小的树。（100Mbit/s,1000M,10000M的const都是1）

    2.3 OSPF的区域划分与优势

    骨干区域，传输区域（Area0，有且只有一个）
    常规区域|Stub区域|NSSA区域（1~43亿）
    ABR(区域边界路由器)：一台路由器宣告到多个Area，其中包含Area0。就会有多个LSDB
    ASBR(自主系统边界路由器)：一台路由器运行OSPF和其他路由获悉方式的协议。比较RIP，静态等。（路由重分发，将其他协议学到的路由，通过OSPF来宣告）

    2.4 OSPF邻居与邻接，MA网络P2P网络建邻居的区别

    Hello。3级Two-way邻居状态Neighbor
    DBD,LSR,LSU,LSAck。8级Full邻接Adjacency

    P2P点到点网络Serial HDLC/PPP（建立邻居，邻接关系一气呵成）
    MA多路访问网络Ethernet Frame Relay（先hell建立邻居，再选举，再所有路由器都会和DP和BDR建立到Full邻接，DROthers之间只会到Two-way邻居关系）

    1. MA网络
      选举：先比较接口的OSPF的优先级(0~255，默认1)，再比较RID，都是大优先；DP指定路由器(一台)，BDR备份指定路由器(一台)，DROthers
      选举时间：是hello时间的4倍。在网络中建立 Two-way后就开始了，选举之后就不变了，是不可以抢占的。后面的优先级大也没有用。
      DR和BDR是接口的概念，不是路由器的概念。
      优先级为0视为弃权选举的资格
      一个MA网络可以没有BDR，但是必须有DR.
      DR/BDR必须能访问该网络中的所有其他节点(Hub&Spoke Spoke之间的互访需要通过Hub中转)

    2.6 OSPF的1-5类LSA

    1. Hello(initial，Two-wait)：建立邻居和维护邻接关系状态
    2. Database Description：LSA的包头，做数据库的同步。DBD(LSA报头，展示数据库里面有那些LSA)；隐式DBD确认（根据RouteID选取主从关系，Master/Slave;通过相同的序列号来既做确认也做回应）; MTU一致性检测,exstarts
    3. Link-State Request(LSA报头，期待你给我请求的完整内容)；等DBD发完之后才开始的。
    4. Link-State Update（完整的LSA）
    5. Link-State Acknowledgment（收到LSU后，会针对LSU中我所收到的LSA来做确认，回复那些LSA我收到了）

    OSPF网络类型：广播类型(需要选举DR和BDR)；点到点类型(不用选举)

    Hello报文要一样才可以建立邻居关系: 报头(版本一致；路由器IP不冲突；直连路由器的Area要一致；认证方式要相同；认证的密钥要一样)，载荷(非点到点网络，掩码需要一样；hello时间要一样；deal时间也要一样；Option的N位和E位的值，常规区域N0和E1, Stub区域N0和E0, NASS区域N1和E0)。

    2.7 邻接关系8级

    down：路由器接口刚被宣告到区域，没有收到报文。
    attempt：(NBMA:非广播多路访问，不支持发送组播报文；P2P NBMA网络)不支持发送组播的接口才会有这个状态。
    init
    tow-way
    exstate：发送Frist DBD。Flag=7，第一个DBD，后面还有DBD，我想成为Master => i=1,m=1,M/S=1
    exchange：Master发送带有LSA报头的DBD，序列号是Frist DBD的序列号+1，收到之后做一个相同序列号相同的确认。完成全部DBD报文的交互。
    loading：LSR，LSU，LSACK
    full：LSR为空时，LSDB完全同步了。

    2.8 LSA类型

    1: Router LSAs：每台路由器都发送一条LSA；区域内路由器传输；(路由器名称，类型，宣告接口的信息link(骨干Link[TransNet,Point-to-Point,Virtual]，stub链路) | )
    2: Network LSAs: DR发送一条LSA；区域内的所有路由器都可以收到；该MA网络中的所有路由器ID，该MA网络的接口掩码。
    3or4: Summary LSAs：ABR发送；跨区域传递；一条3类LSA包含的就是一条完整的区域间路由；
    4：与ASBR同区域的ABR产生发送；跨区域传递；包含该ABR去往ASBR的路径度量值
    5: Autonomous system external LSAs：ADBR发送；跨区域传递；一条外部路由就是一条5类LSA
    6: Multicast OSPF LSA(不重要)
    7: Defined for not-so-stubby areas
    8: External attributes LSA for Border Gateway Protocol(BGP)
    9,10,11: Opaque LSAS(不透明LSA)

    LSA的路由器
    LSA的传输范围
    LSA的包含内容

    Link State ID：该LSA的名字
    ADV(Advertisement)：Router 始发者路由器的路由RID
    命名规则：
    1类LSA：2个字段都是发者路由器的路由RID
    2类LSA：LSID是接口的IP地址；ADV　Router是DR的RID
    3类LSA：LSID是路由的前缀；ADV　Router是ABR的RID（会改变）
    4类LSA：LSID是ASBR的RID；ADV　Router是与ASBR同区域的ABR RID（会改变）
    5类LSA：LSID是路由的前缀；ADV Router是ASBR的RID（不会改变）

    2.9 OSPF邻居关系

    2.9.1 邻居关系建立过程

    理论描述

    实验验证

    1. down: 关闭状态。只是起了OSPF，但是没有任何接口参与协议计算
    2. init：初始化状态。我给邻居发送了hello，但是还没有收到对方的hello，或者收到了对方发送的hello，但是hello neighbor字段没有我的router ID
    3. attempt: 尝试。这个是只有NBMA环境下才会有的一个状态。(non-broadcast Specify OSPF NBMA network; 非广播，在这个环境下只能发送单薄数据，不能发送组播和广播，这个环节下可以发送给多个成员，比如DMVPN场景下)所有这个环节下需要手工指定，思考使用neighbor命令，华为使用peer。
    4. two-way：双边邻居关系。收到了邻居的hello，并且从hello的neighbor字段看到了我自己的router ID,这是双边的邻居关系(三次握手，类似issi中的three ways，防止出现单通)。
    5. exstart：这里做三件事情：1.选举主从；2.确认隐式确认使用的序列号；3.比较MTU(华为默认开启MTU检测，思科默认开启MTU检测)。I=1
    6. exchanage：用来交互DBD。I=0的DBD。Master提供序列号，Slave使用该序列号携带载荷开始隐式确认；数量不够时就恢复M=0的DBD，序列号正常计算，载荷为空。
    7. loading：交互LSA
    8. full：

    DR/BDR：用来同步LSA使用的；
    LSA：描述链路状态信息；
    Master/Slave(主从)：用来做隐式确认使用的，用来交互DBD，仅用于同步LSA之间发送DBD使用。
    DBD: first DBD, 载荷为空，    I(Init)=1    ，M(More)=1,MS(Master Yes)=1。根据Master的序列号进行回复(隐式确认，序列号+1)。
    Slave回复DBD后，Slave进入exchange状态；Master收到DBD后，进入Exchange。
    DP/BDR是接口概念，是一个接口的角色，不是设备的角色；是一个链路内一个DR，一个BDR；一个区域会有多个DR，不确定。
