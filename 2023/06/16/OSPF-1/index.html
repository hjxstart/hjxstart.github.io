    概述

    LSA: Link-State Advertisements

骨干路由器: 有一个接口属于Area0的路由器
内部路由器: 全部接口都只属于非Area0区域的路由器
ABR路由器: 有接口属于Area0，其他区域，Area0有full的邻居
ASBR路由器: 自治系统路由器，任何一台路由器都可以成为ASBR，具有其他方式学习的路由引入OSPF，路由重分发，产生了外部路由进入OSPF

    OSPF报文

    Hello

    周期性发送，用于维护邻居状态。

    Database Description

    用于优化邻接建立的流程，确认双方需要的LSA(除开公共部分)，通过序列号实现隐式确认回复功能

    Link-State Request

    请求包含需要的LSA报头，

    Link-State Update

    对方需要的LAS报头

    Link-State Acknowledgment

    确认LSA报头收到

    OSPF邻接关系建立流程

    Down # 没有收到过任何报文
    Attempt # 不常见 NBMA / P2MP NBMA网络，不会主动发Hello, 需要指定neighbor地址才可以发送。(可以发出单播报文，但是没有收到回应的状态)
    Initial # 通过邻居收到hello，但是neighbor字段没有包含我的路由器ID。(本路由器可以确认收到对方的hello报文，但是本路由器的报文不确定对方是否可收到)
    Two-way # 通过邻居收到hello，neighbor字段有包含我的路由器ID。（双向邻居关系）
    Exstart # 选主从路由器(DBD), I:M:M/S = 111 = 0x07
    Exchange # 主路由器会带动备路由器发起DBD的交互过程，有主路由器发起序列号，备路由器做隐式确认。知道双方的LSA
    Loading # 开始彼此交换对方需要的LSA内容，发送LSR、LSU、LSACK
    Full adjacency # 交换玩LSA，邻接状态建立完毕

    SLA学习

    学习方向

    1. LSA产生: 那些路由器可以产生该LSA?
    2. LSA范围: 产生的LSA谁可以接受到
    3. LSA内容: 产生的LSA包含什么内容

    一类LSA (路由器描述)

    1. LSA产生: 任何路由器都可以产生一类LSA。每台路由器针对它所属的每一个区域会产生一条一类LSA。
    2. LSA范围: 区域内。Area1内的路由器产生的一类LSA，只有Area1内的其他路由器可以收到。
    3. LSA内容: 可分为2类内容。一类是用于介绍该路由器是什么路由器，比如路由器的类型/RID; 第二类信息是该路由器宣告了那些接口进OSPF，和接口连接的链路类型。(P2P网络中，包含邻居的RID，本地接口IP地址；MA网络中，包含DR接口的IP地址，本地接口IP地址）

    二类LSA (MA网络描述)

    1. LSA产生: 由DR(DR是伪节点,DR代表的就是一个完整的MA网络)产生
    2. LSA范围: 区域内。泛洪发给区域内的所有路由器
    3. LSA内容: 包含2类内容。第一类内容是一个列表中包含了MA网络中的每个路由器的RID；第二类是描述了MA网络中所有接口的掩码。

    三类LSA (区域级路由共享)

    1. LSA产生: 由ABR产生，LSA数量 = 要转换的区域内路由的数量。
    2. LSA范围: 整个自治系统。
    3. LSA内容: 就是路由信息(前缀/掩码/度量值/下一跳)
