## 1.1 静态路由两种配置方式

1. 出接口

```bash
# 点对点网络可以直接配置出接口路由
ip route 10.1.1.0 255.255.255.255 gigabitethernet 0/1
```

2. 下一跳

```bash
# 多路访问网络MA建议配置下一跳路由
ip route 10.1.1.0 255.255.255.0 12.1.1.2 # 递归查询
```

`如果2种路由都配置，则需要先配置出接口，再配置下一跳`

## 1.2 静态路由高级特性 permanent 以及安全链路

### 1.2.1 概述

> `permannet`(永久):尽可能保证静态路由保留在路由表中
> 静态路由配置完后，会不会一直保存在路由表中？直连路由肯定会在路由表中。静态路由需要看条件。
> 静态路由和直连路由都消失的场景: 出接口 down 状态
> 静态路由消失的场景: 下一跳不可达(查路由表，没有找到对应的出接口)

### 1.2.2 permannet 的作用

> 永久链路
> 按需链路(Circuit On Demand): PPPoE(宽带连接)\ISDN\BRI
> 专线: MPLS VPN(主专线);ISDN 128k(备份链路,按需链路)
> 备份链路在没有流量的时候接口是 down 的，路由就不会存在。所以当主链路 down 的时候，需要切换到备份链路
> 路由出现 需要 流量走；流量要走 需要 路由出现。
> 为了保证主链路 down 了，备份链路可以立即投入使用，所以需要使用 permannet 参数。
> permannet 参数使用的场景: 涉及到按需链路的时候。配置路由之初，需要保证接口是 up 状态。 接口是 up 是，配置路由表肯定会在路由表中，当配置路由的时候添加 permannet 参数的时候，permannet 参数可以保证就算出接口 down 了，也不会让路由消失，路由会一直存在路由表中。

`不要全部清路由表: clear ip route *`

## 1.3 高级特性 通过静态路由覆盖路由表中 IGP 路由实现 DV 协议的路由过滤

> 相同路由：前缀，掩码完全一样的路由 PK 管理距离(优先级)
> 配置静态路由会覆盖距离矢量路由，导致一些通过距离矢量学习到的路由不会通过距离矢量再发给邻居。

距离矢量路由会转发的路由：

1. 该路由器所有宣告进该协议的接口的直连路由
2. 该路由器通过该协议学习到的已加入路由表的该协议的路由

距离矢量协议 DV：EIGRP, RIP

```bash
system-view
interface loopback 0
ip address 3.3.3.3 32
rip 100
version 2
undo summary
network 1.0.0.0
network 12.0.0.0
display ip routing-table protocol rip
```
